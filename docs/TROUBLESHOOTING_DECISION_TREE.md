## Decision Tree Usage Guidelines

### When to Use Each Tree

1. **System Health Tree**: For general system issues or unknown problems
2. **Performance Tree**: For response time, error rate, or resource usage issues
3. **Service Availability Tree**: When specific services are down or unreachable
4. **Database Tree**: For database-specific connection, performance, or data issues
5. **Integration Tree**: When external service dependencies are failing
6. **Security Tree**: For any security-related incidents or suspicious activity

### Escalation Decision Points

Always escalate when:
- **Issue persists** after following decision tree completely
- **SEV-1 criteria** are met (complete outage, data loss, security breach)
- **Business impact** is significant (revenue loss, customer impact)
- **Multiple systems** are affected simultaneously
- **Root cause unclear** after systematic diagnosis

### Documentation Requirements

When following decision trees:
1. **Document decisions** made at each branching point
2. **Record time spent** on each investigation phase
3. **Note commands executed** and their results
4. **Track escalation** decisions and timing
5. **Update decision trees** based on lessons learned

### Quick Reference Commands

```bash
# Essential diagnostic commands
kubectl get pods -n production
kubectl get events -n production --sort-by='.lastTimestamp' | tail -20
kubectl logs -n production -l app=api --tail=100 | grep ERROR
kubectl top pods -n production
curl -f https://api.insurance-lead-gen.com/health

# Database diagnostics
kubectl exec -n production deployment/postgres -- \
  psql -U postgres -c "SELECT count(*) FROM pg_stat_activity;"
kubectl exec -n production deployment/api -- npm run db:test-connection

# Performance checks
curl -w "@curl-format.txt" -o /dev/null -s https://api.insurance-lead-gen.com/health
kubectl exec -n production deployment/prometheus -- \
  promtool query instant 'rate(http_requests_total{service="api",status=~"5.."}[5m])'
```

### Success Criteria

Each decision tree should lead to:
1. **Clear root cause identification**
2. **Appropriate resolution action**
3. **Successful issue resolution**
4. **Documentation for future reference**
5. **Process improvement opportunities**

Remember: The goal is rapid, systematic problem resolution with minimal customer impact.
