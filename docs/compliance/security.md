# Security

This page provides a high-level view of security controls and operational expectations.

For deeper internal references, see:

- [Security Hardening](../SECURITY_HARDENING.md)
- [Secure Development Guide](../SECURE_DEVELOPMENT_GUIDE.md)
- [Audit Logging](../AUDIT_LOGGING.md)

## Core Controls

- Authentication and RBAC
- Encryption in transit and at rest
- Secrets management and rotation
- Audit logging for sensitive actions
- Monitoring and alerting

## Operational Expectations

- Rotate credentials on a defined schedule
- Use least privilege for users and service accounts
- Treat suspected data exposure as an incident
